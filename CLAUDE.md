# kim-develop

**개발자가 없어진 세상에서 개발자로 살아남기** — Phaser 3 + Vite로 만드는 2D 도트 액션 게임.
주인공은 **김개발**, 최종 보스는 **김흑막**. 모든 텍스트와 주석은 한국어로 쓴다.

## 실행

```
npm run dev     # vite (포트 5173)
```

기본 node가 v14라 vite 7이 뜨지 않는다. `.claude/launch.json`이 mise의 node 24를
직접 가리키고 있으므로, 미리보기는 Bash 대신 preview_start(`kim-develop`)로 띄운다.

개발 모드에서만 콘솔에 `game`, `GameState`, `dev`가 노출된다.
`dev.go(씬키)` / `dev.tp(x,y)` / `dev.lv(n)` / `dev.god()` / `dev.clear()` / `dev.kit()`,
백틱(`` ` ``) 키로 디버그 오버레이 토글. 정의는 [main.js](src/main.js).

## 구조

| 파일 | 역할 |
| --- | --- |
| [config.js](src/config.js) | 타일 크기·전투/스킬/적/보스 수치, 타일 인덱스 `T`, 깊이 `DEPTH` |
| [state.js](src/state.js) | 무기·포션·스킬 정의, 씬을 넘어 유지되는 전역 `GameState` |
| src/art/ | 도트. 아스키 아트 + 팔레트로 그린다 |
| [textures.js](src/textures.js) | art/를 모아 굽는다. 스킬 아이콘·탄·보스는 아직 옛 방식 |
| [MapScene.js](src/scenes/MapScene.js) | 모든 맵 씬의 부모. 맵 생성·전투·아이템·충돌을 전담 |
| [FloorScene.js](src/scenes/FloorScene.js) | 던전 각 층을 설정 객체로 찍어내는 팩토리 |
| [UIScene.js](src/scenes/UIScene.js) | 항상 위에 떠 있는 별도 씬. 하단바·힌트·다이얼로그 |
| src/maps/ | ASCII 아트 문자열로 그린 맵 레이아웃 |

**설계 원칙**

- 수치는 씬에 흩뿌리지 않고 `config.js` / `state.js`에 모은다.
- 새 층을 추가할 땐 씬 클래스를 새로 쓰지 말고 `createFloorScene`에 설정을 넘긴다.
- 맵은 ASCII 아트로 그린다(`src/maps/*.js`의 `parseMap`). 배열 하드코딩 금지.
- 월드 카메라만 `WORLD_ZOOM=2`로 확대하고 UI 씬 카메라는 줌 1로 두어 글자를 선명하게 유지한다.
- 쿨타임은 씬마다 초기화되는 시계 대신 게임 전체를 흐르는 `game.loop.time`을 기준으로 잰다.

## 도트 (src/art/)

스타듀밸리 결을 맞추기 위해 **모든 그림을 16 눈금에 그리고 2배로 구워 쓴다**
([pixel.js](src/art/pixel.js)의 `ART_SCALE`). 타일은 화면에서 32px이지만 도트는
16칸으로 성글게 찍히므로 캐릭터와 배경의 픽셀 굵기가 같아진다.
32 눈금에 그대로 그리면 도트가 잘아져서 느낌이 죽는다.

- 그림은 맵과 같은 방식으로 **아스키 아트**로 그린다. 글자 하나가 도트 하나고
  무슨 색인지는 팔레트가 정한다. `.`과 공백은 투명. 줄 길이가 어긋나거나
  팔레트에 없는 글자를 쓰면 `assertArt`가 즉시 알려 준다.
- 재질마다 기본/그림자/하이라이트 세 톤을 둔다. 단색 채우기 금지.
  외곽선은 검정이 아니라 그 재질을 어둡게 누른 색이다.
- 사람은 [hero.js](src/art/hero.js) 한 벌을 김개발과 추종자가 **팔레트만 바꿔** 같이 쓴다.
  16x32(→32x64)에 앞·뒤·옆 세 자세, 자세마다 걷기 두 프레임.
  걷기는 다리를 갈아끼우고 상체를 한 도트 내려앉히는 반동으로 만든다.
- **김개발 가슴의 사원증은 장식이 아니다.** 거리에서 추종자들이 "저 녀석 명찰을 봐"라고
  외치는 그 명찰이다. 추종자 팔레트는 사원증 색을 옷 색으로 맞춰 두어
  같은 그림인데도 명찰만 사라진다 — 명찰의 유무가 둘을 가르는 표식이다.
- 배경 팔레트는 `warm`(형광등이 살아 있던 사무실) / `cold`(번개가 친 뒤) **두 벌**이다
  ([palette.js](src/art/palette.js)). 같은 도트를 두 번 구워 두고
  씬의 `mood` 게터가 어느 쪽을 쓸지 고른다. 세상이 식는 연출은 이 값만 바꾸면 된다.
- 사무실은 **흰색·회색**이다. 나무로 깔았더니 사무실이 아니라 교실로 보였다.
  '따뜻함'은 색이 아니라 밝기와 살짝 도는 노란기로만 낸다.
- 창문(`T.WINDOW`, 맵 글자 `V`)은 사무실 위쪽 벽에 몰려 있다. **오프닝에서 번개가 치는 자리**다.
  유리 색은 팔레트의 `y`·`Y`라, 번개는 이 두 색만 흔들면 된다.
- 사람과 소품은 y좌표를 그대로 깊이로 써서 아래에 선 쪽이 앞을 가린다(`sortDepth`).
  이펙트는 `DEPTH.effect` 위에 둔다.
- 물리 몸통은 벽에 머리가 걸리지 않게 **발밑만** 잡는다. 그래서 맞는 판정은
  물리 몸통이 아니라 `hitArea`(보이는 몸)로 따로 잰다. 이걸 헷갈리면
  가슴 높이로 날아온 탄이 그냥 통과한다.
- 책상 위 소품은 타일이 아니라 그림이다(`spawnProps`). 맵 글자표를 늘리지 않고 배치를 늘릴 수 있다.

## 컷신 (src/cutscene/)

[CutsceneScene.js](src/cutscene/CutsceneScene.js)가 맵 씬 **위에 얹혀서** 돈다.
넓은 컷은 아래 맵을 그대로 비추고 카메라만 옮기고(하이브리드), 얼굴 컷일 때만
검은 막을 깔고 클로즈업을 띄운다. 그래서 컷마다 배경을 새로 그릴 필요가 없다.

대본은 [scripts.js](src/cutscene/scripts.js)에 한 줄이 한 컷인 배열로 둔다.

- `panel` 화면을 채우는 배경 그림 / `portrait` 얼굴 클로즈업
- `camera` `{x, y, zoom, duration}` — 타일 좌표로 아래 맵을 비춘다
- `who`·`text` 대사. text가 있으면 키를 누를 때까지 기다린다
- `hold` 대사 없는 컷이 머무는 시간(ms)
- `do` 그 순간 세상에 벌어지는 일(동료 사라짐, 번개, 정전)

**`do`는 몇 번 실행해도 같은 결과여야 한다.** Esc로 건너뛰면 남은 컷의 `do`를
몰아서 실행해 세상을 컷신이 끝난 상태로 맞추기 때문이다. 그냥 씬만 닫으면
동료가 안 사라지거나 불이 꺼진 채로 게임이 시작된다.

얼굴은 [portraits.js](src/art/portraits.js)에 24x24로 따로 그린다. 걸어다니는
스프라이트는 눈이 도트 한 칸이라 확대해도 표정이 안 나온다. 반대로 배경은
아스키로 그릴 수 없어([panels.js](src/art/panels.js)) 도형으로 그리되,
좌표를 정수로만 찍고 2배 확대해 도트 굵기를 타일과 맞춘다.

- `title` 화면 한가운데 크게 뜨는 글자(엔딩의 `theEnd();`와 제목)
- `figures` 배경 위에 세우는 인물. `cropBottom`으로 하반신을 잘라 상반신만 남길 수 있다

이미 본 컷신은 `GameState.watched`에 남아 맵으로 되돌아와도 다시 돌지 않는다.

**엔딩은 장소가 둘이라 컷신도 둘이다.** 비밀공간에서 `endingSecret`이 돌고
화면이 하얘지면 `GameState.ending`을 세운 채 사무실로 넘어가고,
사무실이 그 깃발을 보고 `endingOffice`를 이어 튼다.
김흑막은 쓰러져도 사라지지 않는다 — 엔딩에서 자기 입으로 진실을 말해야 하기 때문이다.
컷 안의 좌표는 보스의 현재 위치가 아니라 **처음 서 있던 자리**를 기준으로 잡는다.
보스는 싸우는 동안 방을 떠돌아서, 어디서 쓰러졌느냐에 따라 구도가 매번 달라진다.

## 이펙트 ([effects.js](src/effects.js))

연출은 판정과 완전히 분리돼 있다. 여기서 무엇을 그리든 맞고 안 맞고는
`MapScene.hitArea`가 정하므로 **연출은 마음껏 과장해도 된다**.

- 평타(Q)는 들고 있는 키보드를 무기 색으로 물들여 호를 그리며 휘두른다.
  맨손이면 휘두른 자취만 남는다 — 없는 키보드를 그릴 수는 없으니.
- 스킬은 `SKILLS[].color`를 써서 시전 순간 몸에 빛이 돌고 발밑에 고리가 퍼진다.
- 포션은 `POTIONS[].color`로 알갱이가 몸을 타고 오른다.
- 확대되는 도형은 그래픽을 **원점(0,0)에 그리고 객체를 옮겨** 놓는다.
  월드 좌표에 직접 그리면 확대할 때 제자리에서 커지지 않고 화면 밖으로 밀려난다.

## 게임 흐름

```
BootScene → OfficeScene → StreetScene → LobbyScene → Floor2Scene → ChairmanScene → SecretScene(보스)
```

씬 키 `SecretScene`의 구현 파일은 `BossScene.js`다(파일명과 키가 다르다).

각 구간의 관문:

- **사무실** — 튜토리얼. 아이템 3개를 다 주워야 나갈 수 있다. 맨손으로는 못 나간다.
- **거리** — 첫 전투. 2킬 하면 `bug`(버그 소환) 스킬이 열린다. 전멸시켜야 건물로 들어간다.
- **던전 각 층** — 전멸 + 필수 아이템 획득을 둘 다 만족해야 다음 층으로.
  회장실은 전설의 키보드(`chairman-legend`)를 들어야 차원문이 열린다.
- **비밀공간** — 잡몹을 다 잡으면 셔터가 열리고 김흑막 전이 시작된다.

## 규칙

- HP/DP 100 시작이 아니라 **HP 20으로 시작**한다(야근으로 체력이 바닥난 설정). 버그가 아니다.
- 무기는 획득 즉시 장착되고 되돌릴 수 없다. 다이소 → 기계식 → 전설.
- 스킬은 Q(평타, DP 0) / W(버그) / E(소리어택) / R(병풍코딩). 레벨 1/1/3/5에 열리고
  `bug`만 레벨이 아니라 **진행(2킬)** 으로 열린다.
- 만렙 5. 경험치는 HTML 25, CSS 45.
- HP가 0이 되면 죽지 않고 2.5초간 쓰러졌다가 HP 20까지 회복하며 부활한다.
- 아이템 획득 여부는 `GameState.picked`에 남는다. 포션을 다 써도 관문이 다시 잠기지 않는다.

## 현재 상태

**오프닝부터 엔딩까지 처음부터 끝까지 이어진다.** 김흑막을 쓰러뜨리면
비밀공간 엔딩 → 사무실 엔딩 → `theEnd();` → 제목으로 끝난다.

`config.js`의 스킬 DP 소모량과 위력은 밸런싱 전 임시값이라고 주석에 명시돼 있다.

도트는 김개발·추종자·김흑막·HTML의 신·타일·소품·아이템·얼굴까지 새 방식으로 옮겼다.
아직 남은 것:

- **스킬 아이콘과 CSS 탄**이 옛 방식(`textures.js`)이라 도트가 잘다.
- **타일셋이 모든 맵 공유**다. 사무실 카펫이 던전에도 그대로 깔린다.
  맵마다 다른 분위기를 내려면 타일셋을 맵별로 쪼개야 한다.
- 벽이 한 종류라 위·아래·모서리 구분이 없다. 벽 타일을 늘리면 방이 훨씬 방다워진다.
- 사람 스프라이트가 커진 만큼 **전투 사거리 수치는 다시 봐야 한다**(`COMBAT`).
- 소리가 없다.
